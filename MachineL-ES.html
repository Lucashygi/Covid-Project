Machine Learning on Espírito Santo's Deaths


This part is responsible for presenting the evolution in some results of Machine Learning algorithms based in the state of Espírito Santo's (Brasil) datasets, presenting data from the very first case in the state until the end of the months of March, April, May, and June 16, 2020, and using only information referring to biological and symptomatic conditions such as:

  • Age
  • Sex
  • Ethnicity
  • Fever
  • Shortness of breath (Dyspnea)
  • Cough
  • Coryza
  • Sore Throat
  • Diarrhea
  • Headache
  • Lung Diseases
  • Cardio Diseases
  • Kidney Diseases
  • Diabetes Disease
  • Obesity Disease
  • Smoking
  • Hospitalized confirmed***
  • Confirmed Death

    • ***Not included in June

The results were found through algorithms and filters applied to WEKA Workbench.

The filters were:

  • No Filtered
  • Class Balancer
  • SMOTE

No filtered -> With no filters.
Class Balancer -> It balances the weight of the classes, equaling them out.
SMOTE -> It builds values in the lower class based in its neighboors features. (To set both classes equal the filter was applied in every case using SMOTE).

The algorithms used were the following:

  • Decision Table
  • Logistic
  • Naïve Bayes
  • Simple Logistic
  • SPegasos
  • Support Vector Machine (SVM)
  • REP Tree

SPegasos and SVM don't have implementation using Class Balancer without a Cost Matrix
